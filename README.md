## Descrição do caso escolhido

O projeto foi desenvolvido para a Planet Music, uma loja especializada na venda de instrumentos musicais e acessórios. A proposta foi criar um site institucional moderno, organizado e responsivo, permitindo que clientes conheçam a loja, seus principais produtos e encontrem facilmente formas de contato.

---

## Necessidade identificada

A Planet Music não possuía uma presença digital organizada para apresentar seus produtos e facilitar o contato com novos clientes. Atualmente, muitas pessoas procuram informações sobre lojas pela internet antes de realizar uma compra, tornando essencial a existência de um site que transmita credibilidade e profissionalismo.

Diante dessa necessidade, foi desenvolvido um site simples, intuitivo e acessível, capaz de divulgar os produtos da loja e aproximar clientes da empresa.

---

## Público-alvo

O site foi pensado para atender:

- músicos iniciantes;
- músicos profissionais;
- estudantes de música;
- professores;
- igrejas;
- escolas de música;
- pessoas interessadas em adquirir instrumentos musicais.

Esses usuários precisam encontrar rapidamente informações sobre os produtos oferecidos, formas de contato, localização da loja e canais de atendimento.

---

## Objetivo do site

O principal objetivo da página é apresentar a Planet Music de forma profissional, permitindo que os visitantes conheçam a loja, visualizem seus principais produtos e sejam incentivados a entrar em contato para solicitar informações ou realizar um orçamento.

---

## Processo de desenvolvimento

Inicialmente realizamos uma análise da Planet Music para identificar suas necessidades de divulgação e comunicação com os clientes. Após o diagnóstico, definimos quais informações seriam mais importantes para o público-alvo.

Em seguida, elaboramos a estrutura da página utilizando HTML semântico e organizamos as seções do site. Posteriormente, desenvolvemos a identidade visual utilizando CSS e implementamos a responsividade com Bootstrap e Media Queries.

Após a construção da interface, adicionamos funcionalidades em JavaScript para melhorar a experiência do usuário. Por fim, realizamos testes em diferentes tamanhos de tela, verificamos os links, imagens e o funcionamento geral do site antes da publicação no GitHub Pages.

---

# Principais decisões do projeto

Durante o desenvolvimento foram tomadas algumas decisões importantes:

- utilização de HTML semântico para melhorar a organização e acessibilidade;
- criação de uma identidade visual utilizando cores escuras e detalhes em amarelo, remetendo ao universo musical;
- organização das informações em seções bem definidas;
- utilização de imagens ilustrativas para facilitar a identificação dos produtos;
- desenvolvimento de uma navegação simples e intuitiva;
- implementação de um layout totalmente responsivo;
- aplicação de boas práticas de acessibilidade, como textos alternativos (alt), atributos ARIA e contraste adequado entre texto e fundo.

---

# Funcionalidade em JavaScript

Foi implementado um menu responsivo (hambúrguer) para dispositivos móveis.

Ao clicar no botão, o menu é aberto ou fechado dinamicamente. Além disso, após selecionar uma opção da navegação, o menu é fechado automaticamente.

Também foi utilizado o atributo `aria-expanded`, melhorando a acessibilidade para usuários que utilizam leitores de tela.

Essa funcionalidade foi escolhida por melhorar significativamente a experiência de navegação em dispositivos móveis.

---

# Uso de Bootstrap

O Bootstrap foi utilizado para auxiliar na construção do layout responsivo do projeto.

Foram utilizadas classes como:

- container;
- row;
- col;
- sistema de Grid;
- utilitários de espaçamento.

Seu uso facilitou a adaptação da página para diferentes resoluções sem substituir a construção manual realizada com HTML e CSS.

---

## Testes realizados

Foram realizados testes em diferentes tamanhos de tela utilizando as ferramentas de desenvolvedor do navegador para verificar a responsividade.

Também foram testados:
- Funcionamento do menu hambúrguer;
- Exibição da mensagem de confirmação do formulário;
- Carregamento correto das imagens;
- Navegação pelos links internos;
- Legibilidade dos textos em dispositivos móveis;
- Navegação básica por teclado;
- Funcionamento do site após a publicação no GitHub Pages.
